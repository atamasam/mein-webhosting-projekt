Software-Architektur von Webanwendungen

Grundlegend wird bei Software-Architekturen für Webanwendungen zwischen Client-seitiger Programmierung und Server-seitiger Programmierung unterschieden. Bei beiden Arten gibt es Vor- und Nachteile.

Client-seitig: Rich – thick Client

Beschreibung: Anzeige und Dokument im Webclient -> Interaktion mit dem Webserver

  • Vorteile: Technische Umgebung des Benutzers kann berücksichtigt werden
  • Nachteile: Verarbeitung des Programmcodes kann vom Nutzer unterbunden werden

Server-seitig

Beschreibung: Anzeige und Dokument im Webclient -> Interaktion mit dem Webserver. Der Webserver interagiert mit Java Servlets, CGI-Anwendungen und PHP-Anwendungen.

  • Vorteile: Man ist nicht darauf angewiesen, dass der Nutzer die clientseitige Verarbeitung einschaltet
  • Nachteile: Technische Umgebung des Benutzers wird nicht berücksichtigt

Erweiterung um Programme oder Skripte

Beschreibung: Anzeige und Dokument im Webclient -> Interaktion mit dem Webserver. Der Webserver interagiert mit Java Servlets, CGI-Anwendungen und PHP-Anwendungen.

  • Vorteile: Vorteile von Client- und Serverseitiger Verarbeitung können genutzt werden

Übersetzung von Programmiersprachen

  • Ein Übersetzer transformiert ein Programm der Quellsprache in ein gleichbedeutendes Programm der Zielsprache
  • Die meisten Programmiersprachen werden übersetzt
  • Zielsprache ist meist eine Maschinensprache eines Prozessors oder einer abstrakten Maschine (z.B. JVM)
  • Übersetzte Sprachen haben eine stark ausgeprägte statische Semantik

Interpretation von Sprachen

  • Ein Interpretierer liest und führt ein Programm aus
  • Strikt interpretierte Sprachen haben einfache Struktur und keine statische Semantik
  • Bindungs- und Typregeln werden erst zur Ausführung geprüft
  • Interpretierer können auf jedem Rechner verfügbar gemacht werden
  • Interpretation kann langsamer sein als die Ausführung von übersetzten Programmen

Skriptsprachen

  • Scripting: Zusammensetzen von Kommandos für kleine Aufgaben
  • Anforderungen: einfache Aufgaben lösen, existierende Funktionen nutzen, Textverarbeitung, gute Verfügbarkeit

Zweck von Skriptsprachen

  • Einfache Sprachen mit wenigen Konstrukten und Regeln
  • Dynamische Typprüfung
  • Interpretiert, ohne vorherige Übersetzung ausführbar
  • Leicht erlernbar
  • Sprachstile: imperativ, objektorientiert

JavaScript

  • Relativ einfache Skriptsprache
  • Speziell für HTML entwickelt
  • Code wird über das <script>-Tag direkt im HTML-Code eingebunden
  • Im Web-Browser interpretiert
  • Basis für dynamisches HTML (DHTML) und Web 2.0

Grundlegende HTML-Datei

HTML-Dateien sind durch Tags aufgebaut, die die Struktur definieren. Schlüsselwörter werden in spitzen Klammern geschrieben, z.B. <html>, <body>. 


  <!DOCTYPE html>
  <html>
  <head>
    <title>HTML Beispiel</title>
  </head>
  <body>
    <p>
    Das ist HTML!
    </p>
  </body>
  </html>

Das abschließende </html> beendet die HTML-Datei.

JavaScript


  <script type="text/javascript">
      // JavaScript Code
  </script>

Eine Möglichkeit in JavaScript ist das Ausgeben einer Meldung mit der Methode alert. 


  <script type="text/javascript">
      alert("Das ist JavaScript!");
  </script>

Beim Aufrufen der Website wird eine Meldung mit dem Text „Das ist JavaScript!“ angezeigt, die mit Ok bestätigt werden kann.

Weiterhin ist es möglich in JavaScript zu rechnen. Dazu benutzt man für die Ausgabe wieder die alert-Funktion, ersetzt aber den Text durch eine mathematische Operation. 


<script type="text/javascript">
    alert(17 + 5 * 3);
</script>

Als Ergebnis würde hier 32 ausgegeben werden. Im Gegensatz zum ersten Beispiel wird die Rechnung nicht in Anführungszeichen angegeben, da der Computer so erkennt, dass es sich nicht um einen normalen Text handelt.

Häufig soll JavaScript-Code nicht direkt beim Aufrufen der Website ausgeführt werden, sondern erst als Reaktion auf eine Usereingabe. Hier bietet sich z.B. ein Input vom Typ Knopf an, der eine Meldung auslöst. 


<form>
   <input type="button" value="Test" onclick="alert('Hallo!')">
</form>

Sobald der User also auf den Knopf klickt, wird „Hallo!“ ausgegeben.

Ein wichtiger Punkt beim Programmieren ist das Kommentieren des Codes. Dies hilft einerseits dem Programmierer selber später dabei, seinen Code auch nach längerer Pause wieder zu verstehen. Außerdem erleichtert es anderen Mitarbeitenden das Verständnis des Programms.

In JavaScript wird ein einzeiliger Kommentar mit zwei Schrägstrichen eingeleitet: 


// Dies ist ein Kommentar

Mehrzeilige Kommentare starten mit /* und enden mit */: 


/* Dies ist ein mehrzeiliger Kommentar.
Alles was zwischen den Begrenzern steht, wird nicht
im Browser angezeigt. */

Abschließend sollte auch bedacht werden, dass bestimmte Browser/User JavaScript deaktiviert haben, was die Funktionalität einer Seite beeinflussen kann. Hierzu kann der <noscript>-Tag benutzt werden, der Nutzern entweder signalisiert, dass diese Website JavaScript benutzt, oder auf eine alternative Website verweist, die auch ohne funktioniert.
