Vorlesung 2: Einführung in die Datenübertragung und Netzwerkprogrammierung (07.04.2025)

In der zweiten Lektion haben wir uns mit dem Konzept der Datenübertragung beschäftigt und untersucht, wie Informationen in Java gespeichert werden können, um sie für andere Programme zugänglich zu machen. Ein besonderer Fokus lag auf dem Schreiben von Strings in eine Textdatei.

Einführung in die Dateiverwaltung

Zur Speicherung von Daten wird der FileWriter verwendet. Mit diesem kann eine Datei geöffnet oder erstellt werden, und Daten in die Datei geschrieben werden.

Für den Zugriff auf Dateien wird die File-Klasse verwendet, mit der es möglich ist, Objekte zu erzeugen, die eine Datei oder einen Pfad repräsentieren. Auch das Erstellen neuer Verzeichnisse ist mit dieser Klasse möglich.

Zum effizienteren Schreiben von Daten wird der BufferedWriter genutzt. Dieser sammelt Daten in einem Puffer und schreibt sie erst dann in die Datei, wenn der Puffer voll ist.

Ähnlich funktioniert das Lesen von Dateien mit einem FileReader und einem BufferedReader. Der BufferedReader liest Zeile für Zeile und verarbeitet jede Zeile nacheinander.

Übung: Textdatei kopieren

In der Übung haben wir eine Textdatei in Java „kopiert“, indem wir den Inhalt einer .txt-Datei mit einem BufferedReader und einem FileReader ausgelesen und anschließend mit einem BufferedWriter und einem FileWriter in eine neue Datei geschrieben haben. Dabei wurden Zeilenumbrüche und andere Formatierungen korrekt übernommen.

Netzwerkprogrammierung mit Sockets

Im weiteren Verlauf der Vorlesung haben wir uns mit der Netzwerkkommunikation über Sockets beschäftigt. Es wurde erläutert, wie Daten zwischen einem Client und einem Server ausgetauscht werden können. Der Client muss dabei den Server kennen, während der Server alle verbundenen Clients verwaltet.

Schritte zur Verbindung:

  • Die Verbindung zwischen dem Server und dem Client herstellen.
  • Nachrichten an den Server senden.
  • Nachrichten vom Server empfangen.

Sockets in Java

Sockets stellen die Netzwerkverbindung zwischen zwei Geräten dar. In Java wird dafür die IP-Adresse und eine TCP-Portnummer verwendet. Bestimmte Portnummern sind für Dienste wie HTTP oder FTP reserviert, während Ports zwischen 1024 und 65535 in der Regel frei gewählt werden können, jedoch mit dem Systemadministrator abgestimmt werden sollten.

Zum Lesen von Daten aus dem Socket wird ein InputStreamReader verwendet, der mit einem BufferedReader verbunden wird. Zum Schreiben in den Socket wird ein PrintWriter eingesetzt, der Nachrichten an den Server sendet.

Übung: Tipps des Tages austauschen

In dieser Übung haben wir einen Client und einen Server programmiert, die „Tipps des Tages“ austauschten. Die Tipps wurden als Zeichenketten auf dem Server gespeichert und vom Client abgerufen. Der Client gab diese dann in der Konsole aus.
